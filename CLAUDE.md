# Evening Checkout Journal — Folder Instructions

## Purpose

This folder stores Rick's evening reflection journal entries, generated by the `checkout` CLI app. Entries follow a structured 5-question template focused on presence, joy, values, pride, and letting go.

---

## File Organization

### Folder Structure

```
~/kb/journal/
├── YYYY/
│   └── MM/
│       └── YYYY-MM-DD-checkout-v1.md   <- individual entries
├── index.md                             <- auto-generated index (do not edit manually)
└── CLAUDE.md                            <- this file
```

### Naming Convention

Entries follow strict naming: `YYYY-MM-DD-{templateId}.md`

- Example: `2026-03-16-checkout-v1.md`
- One entry per day per template
- Stored in `YYYY/MM/` subfolders

---

## Entry Format

Each entry follows the `checkout-v1` template:

```markdown
## How present do you feel right now?
[1-10 rating]

## Your joy-moment
[One specific moment of joy from the day]

## Think of your values
[Action taken today aligned with a value]

## Something I'm proud of
[One thing from today]

## What do you decide to let go of?
[Something released before sleep]

---

**Metadata**
- Created: [ISO timestamp]
- Template: checkout-v1
- Version: 1.0
```

## Working with These Files

### Reading entries

- The `index.md` file is the best entry point — lists all entries grouped by month with presence scores
- Individual entries are plain markdown, human-readable as-is

### Querying patterns

When asked to surface patterns or insights from journal entries, look for:

- **Presence score trends** (the 1-10 rating in each entry)
- **Recurring themes** in joy moments, values actions, or pride entries
- **What Rick lets go of** — common sources of stress or frustration
- **Values in action** — which values come up most often and how

### Adding entries

Do not manually create entries in this folder. Entries are created by the `checkout` CLI or via the Alfred Telegram bot (`/checkout`). If asked to create one directly, follow the naming convention and format exactly.

### Index regeneration

`index.md` is auto-generated by `checkout index`. Do not manually edit it. If it's out of date, suggest running `checkout index` in the terminal.
